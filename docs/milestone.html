CS 184: Computer Graphics and Imaging, Spring 2019

Milestone Report

Alan Nguyen, Jared Shao, Vincent Tantra

Overview

As a refresher, our final project is exploring environment rendering in the VR space, using primarily Unity for development and Google Cardboard API for porting our VR application into an iOS phone.

Accomplishments

Due to our lack of experience in VR, we both faced a preliminary problem of actually learning how to use Unity and the Google Cardboard API. What this meant was that we've dove through myriads of documentation and tutorials for both platforms. After being able to port an online demo on Unity into one of our phones through the Google Cardboard API, we are now comfortable enough to navigate the Unity space and begin developing the inner workings of this project. For now, we have a simple scene with various features implemented, including a camera, a scene, and some objects/shapes.

Another problem we are currently facing with this project revolves around optimization. Since we are projecting our VR environment into mobile phones, we ultimately have to make some cutbacks. For instance, we have to implement "baked" lighting for global illumination as opposed to Precomputed Realtime GL Lighting in order to accomodate for mobile hardware. To summarize the presented link, we have a tradeoff of the ability to render lights overtime for increased performance and less memory usage. More research is currently underway for being able to optimize our project for mobile phones.

Schedule

To map out our progress, we have the current workplan on a week-to-week basis, starting from this week. This schedule is similar to our project proposal schedule; our team is a bit aligned with the original schedule.

  1. Play around with more of Unity. Construct the main scene and begin implementing global illumination code. Optimistically, be complete with rendering code and be able to successfully port this draft to Google Cardboard.
  2. Debug and optimize all code drafted from the previous week. Finish up any remaining implementation details, with some specifics including: scene changes, targeted lighting algorithms, performance. Have this draft at the minimum ready for poster session.
  3. Extend this app to include more functional and enjoyable features, not limited to Unity XR and more interaction with Google Cardboard. Details regarding project extension are contingent to what we've accomplished and how much time is left before the final deadline.
